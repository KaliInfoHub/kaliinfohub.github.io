What is Wireshark?

Wireshark is a network protocol analyzer used to capture and inspect packets in real time. It's useful for debugging, monitoring, and analyzing network traffic. As one of the most powerful tools in a network security professional's arsenal, Wireshark allows you to examine data from a live network or from a capture file on disk.

Key Features

  • Deep inspection of hundreds of protocols, with more being added all the time
  • Live capture and offline analysis
  • Multi-platform: Runs on Windows, Linux, macOS, and other Unix-like systems
  • Rich VoIP analysis
  • Read/write many different capture file formats
  • Decryption support for many protocols including IPsec, ISAKMP, Kerberos, SNMPv3, SSL/TLS, WEP, and WPA/WPA2
  • Coloring rules can be applied to the packet list for quick, intuitive analysis

Installation in Kali Linux

Wireshark comes pre-installed in Kali Linux. You can verify the installation or update to the latest version using:

sudo apt update sudo apt install wireshark

To run Wireshark with root privileges (required for capturing on most interfaces):

sudo wireshark
Security Tip: Running Wireshark as root can be a security risk. Consider setting up dumpcap with appropriate permissions instead: sudo dpkg-reconfigure wireshark-common sudo usermod -a -G wireshark $USER Then log out and back in for changes to take effect.

Basic Usage

Select a network interface and start capturing. Use filters to narrow down traffic, e.g.:

http, ip.addr == 192.168.1.10, tcp.port == 80

Starting a Capture

  1. Launch Wireshark
  2. From the home screen, select the network interface you want to monitor
  3. Click the shark fin icon in the toolbar to begin capturing packets
  4. To stop capturing, click the red square icon

Display Filters

Use display filters to view only packets that match specific criteria:

Filter Description
ip.addr == 192.168.1.1 Show only packets with this IP address as source or destination
ip.src == 192.168.1.1 Show only packets from this source IP
tcp.port == 443 Show only packets using TCP port 443 (HTTPS)
http.request Show only HTTP request packets
dns Show only DNS packets
icmp Show only ICMP packets

Combining Filters

Use logical operators to create more complex filters:

http && ip.addr == 192.168.1.10 # HTTP traffic from/to 192.168.1.10 http || dns # Either HTTP or DNS traffic !(arp || icmp) # Everything except ARP and ICMP tcp.port == 80 || tcp.port == 443 # HTTP or HTTPS traffic

Capture Filters

Unlike display filters, capture filters are applied before packets are captured, reducing resource usage:

host 192.168.1.1 # Capture traffic only for this host net 192.168.0.0/24 # Capture traffic for this network port 53 # Capture only DNS traffic not port 3389 # Ignore RDP traffic tcp port 80 and not host 192.168.1.1 # HTTP traffic except from/to 192.168.1.1

Analyzing Packets

Wireshark divides its interface into three main panes:

  1. Packet List: Shows a summary of each packet including timestamp, source, destination, protocol, and info
  2. Packet Details: Shows the protocols and protocol fields of the selected packet in a tree view
  3. Packet Bytes: Shows the raw data of the packet in hex and ASCII

Following TCP Streams

To view the entire conversation in a TCP connection:

  1. Right-click on a TCP packet
  2. Select "Follow" → "TCP Stream"

This is particularly useful for analyzing HTTP, FTP, SMTP, and other text-based protocols.

Using Statistics

Wireshark offers various statistical tools under the "Statistics" menu:

  • Protocol Hierarchy: Shows distribution of protocols
  • Conversations: Lists all connections between endpoints
  • Endpoints: Lists all detected network hosts
  • I/O Graph: Shows packet rate over time

Practical Example

Capture HTTP credentials sent in plaintext and analyze the session behavior between client and server.

Step-by-step Walkthrough

  1. Start Wireshark capture on your network interface
  2. Apply the display filter: http.request.method == "POST"
  3. Generate traffic by logging into a non-HTTPS website
  4. Look for POST requests in the packet list
  5. Select a POST request packet and expand the HTTP protocol in the packet details pane
  6. Look for form data or authorization headers that may contain credentials
  7. Right-click on the packet and select "Follow HTTP Stream" to see the full request and response
Ethical Reminder: Only perform packet analysis on networks you own or have explicit permission to monitor. Capturing credentials on public networks without authorization is illegal in most jurisdictions.

Advanced Features

Decrypting SSL/TLS Traffic

To decrypt HTTPS traffic in Wireshark, you need either:

  • The server's private key (for non-ephemeral key exchange)
  • Pre-master secrets from the client (using browser or OS keylog features)

Configure in: Edit → Preferences → Protocols → TLS → (Pre)-Master-Secret log filename

Expert Information

Access via: Analyze → Expert Information

Command Line Usage

Wireshark includes tshark, a command-line version:

tshark -i eth0 -f "port 80" -w capture.pcap # Capture HTTP traffic to file tshark -r capture.pcap -Y "http.request" # Read file, filter for HTTP requests tshark -i eth0 -T fields -e ip.src -e ip.dst # Display only source and dest IPs

Troubleshooting

No Interfaces Available

If no capture interfaces appear in Wireshark:

  • Verify you have sufficient permissions (run as root or add user to wireshark group)
  • Check if interface is up: ip link show
  • Ensure capture library is installed: apt install libpcap0.8

High CPU Usage

If Wireshark consumes too many resources during capture:

  • Use more specific capture filters to reduce packet volume
  • Disable name resolution (Edit → Preferences → Name Resolution)
  • Consider using tshark with output to file instead

Protocol Decoding Issues

If Wireshark doesn't correctly decode a protocol:

  • Check if the protocol runs on non-standard ports (Edit → Preferences → [Protocol] → Ports)
  • Try enabling heuristic dissection (Edit → Preferences → Protocols → [Protocol])
